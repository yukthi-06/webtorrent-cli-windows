create a new github action YML to generate a windows EXE for x64

# Goal Description
Create a new GitHub Actions workflow file `.github/workflows/build-windows-exe.yml` that uses `pkg` to package the command-line utility into a standalone Windows x64 executable (`webtorrent-cli-windows.exe`), and uploads the executable as a run artifact.

## User Review Required
No major architectural changes or breaking changes are introduced. The workflow will run manually (`workflow_dispatch`) or on tag/release creation.

## Proposed Changes

### GitHub Actions Workflow

#### [NEW] [build-windows-exe.yml](file:///x:/Projects_X/0_Active/C_NodeJs/webtorrent-cli-windows/.github/workflows/build-windows-exe.yml)
Create a new workflow with the following specifications:
- Trigger: `workflow_dispatch` (manual triggers) and `release` (when a new release is published).
- OS: `ubuntu-latest` (fast and sufficient for cross-compilation with `pkg`).
- Steps:
  1. Checkout code.
  2. Setup Node.js (matching version engine, e.g. Node 20).
  3. Install dependencies.
  4. Run `npx pkg bin/cmd.js --targets node18-win-x64 --output dist/webtorrent-cli-windows.exe` (or Node 20 target depending on preference). We will use `node18-win-x64` or `node20-win-x64`. Let's target `node18-win-x64` as indicated in the documentation, or support input choices.
  5. Upload the compiled executable (`dist/webtorrent-cli-windows.exe`) as a GitHub Action run artifact.
